# ARES — ISO/IEC 27001:2022 (ISMS) & ISO/IEC 42001:2023 (AIMS) Application Analysis

> **Status:** Draft for review · **Nature:** Documentation only — no build authorized.
> **Scope of analysis:** the shipped ARES application (ASP.NET Core 8 API + PostgreSQL +
> React SPA; Entra ID sign-in, app-only Microsoft Graph, Teams/email release, deterministic
> severity) **plus** the [CANDIDATE] agentic layer described in
> `docs/agent-design-and-threat-model.md` and doc 01.
> **Deployment context** (per `00-index.md`): on-prem/containerized ARES, Microsoft-centric,
> model hosted **in-tenant** (Azure OpenAI), PostgreSQL, jurisdictions Switzerland + Nordics.

This document maps ARES to **ISO/IEC 27001:2022** (Information Security Management System,
including Annex A's 93 controls in the 2022 four-theme structure) and **ISO/IEC 42001:2023**
(AI Management System). Findings use the index vocabulary — **Met / Partially met / Gap /
N/A** — and cite `FR-*` / `NFR-*` IDs from the doc-01 scheme (`00-index.md` §2). Where a
control depends on a capability that is not in the shipped app, the row is tagged
**[CANDIDATE]**; shipped behaviour is tagged **[SHIPPED]**.

> **Standing caveat that colours nearly every "Met" below.** The shipped API enforces Entra
> JWT validation **only when a tenant is configured**, and even then a valid token is required
> only when `Ares:AllowDemoAuth` is **false**. The default posture (`AllowDemoAuth = true`,
> empty Entra section in `appsettings.json`) runs the API **open** — no authentication, no
> authorization — and CORS falls back to "any origin". This is deliberate pilot/demo ergonomics
> (`Program.cs` lines 26-49, 95-104) but means most access-control controls are **Met by design
> intent / Partially met in code** until the pilot-mode toggle is closed for production. Every
> access-control finding is qualified accordingly.

# PART A — ISO/IEC 27001:2022 (ISMS)

## A.1 ISMS context (Clause 4)

**Purpose of the ISMS in ARES's world.** ARES is *both* an object protected by the ISMS
(a web+AI application processing incident data and directory PII) *and* an instrument of the
ISMS — it is the tooling through which the Birgma/Biltema group runs **information security
incident management** (Annex A.5.24-A.5.28). The analysis therefore treats ARES twice: as a
system to be secured, and as a control that supports the wider organizational ISMS.

### Scope statement (candidate)

> The ISMS scope for ARES covers the ARES incident-command platform — its ASP.NET Core API,
> PostgreSQL datastore, React SPA, the [CANDIDATE] agent orchestrator/Teams bot/M365 Copilot
> agent, and the in-tenant Azure OpenAI model — together with the incident records, directory
> principals/PII, communications, evidence and audit data they process, and the integrations
> to Microsoft Graph (Teams, mail, Entra logs), ServiceDesk Plus Cloud (EU DC) and signal
> sources. Deployment is on-prem/containerized within the Birgma/Biltema Microsoft tenant,
> operating for Switzerland and the Nordics.

### Risk-assessment approach (Clause 6.1 / 8.2-8.3)

- **Method:** asset-based + threat-scenario risk assessment. The **STRIDE model and top-5
  risks already produced** in `docs/agent-design-and-threat-model.md` §7 form the seed risk
  register; doc 21 (MITRE ATT&CK/ATLAS + Zero Trust) extends adversary coverage. Each risk
  should be logged as `RISK-NNN` with likelihood × impact, owner, and treatment.
- **Criteria:** highest-impact asset = **outbound communications + approval authority**;
  confidentiality of **directory PII** and **incident data**; integrity of **timeline/RCA/audit**.
- **Treatment** maps to Annex A controls (SoA below) and to `NFR-SEC-*` / `NFR-PRV-*` /
  `NFR-AI-*` requirements. Residual risk accepted by `STK-05` with `STK-06` sign-off on PII.
- **AI-specific risk** (prompt injection, hallucinated RCA, over-autonomy, excessive agency)
  is assessed under 42001 (Part B) and cross-referenced here where it is also an infosec risk.

# PART B — ISO/IEC 42001:2023 (AI Management System)

Scope of the AIMS = the **ARES agentic layer** (investigate · coordinate · maintain timeline ·
draft communications · draft RCA) and its in-tenant model, per `docs/agent-design-and-threat-model.md`
and doc 01 area `AGT`. ARES's AI role is a **copilot, not autopilot**: the model **proposes**,
humans **commit**; the LLM is **out of the act plane**; severity is **rule-computed**.
